

6.1.7 Práctica de Laboratorio - Análisis de Vulnerabilidades de Sitios Web
Tareas de Soluciones de Seguridad Protego

Como vio con el análisis de seguridad de la red, los escáneres de vulnerabilidades son muy útiles en las pruebas de penetración. No solo descubren vulnerabilidades, sino que, con frecuencia, proporcionan información de referencia que es útil para comprender cómo se explotan las vulnerabilidades y cómo se pueden mitigar las amenazas.

Un escáner de sitios web generalmente se centra en detectar vulnerabilidades en la arquitectura del sitio web, la configuración del servidor y los recursos del sitio. También puede detectar vulnerabilidades en las aplicaciones web. En esta práctica de laboratorio, aprenderá a utilizar el escáner de vulnerabilidades web de código abierto conocido como Nikto. Es una gran herramienta y puede agregarla a su caja de herramientas de pruebas de penetración.

A medida que continuamos interactuando con Pixel Paradise, escanearán su sitio web en busca de vulnerabilidades. Nikto es un escáner “ruidoso”, pero eso está bien porque se ha indicado al personal de seguridad que permita nuestros escaneos en momentos preestablecidos. Más adelante en la prueba, utilizaremos herramientas más furtivas, al azar, para determinar si el equipo de seguridad de Pixel Paradise detecta y bloquea nuestros análisis.

En esta práctica de laboratorio se cumplirán los siguientes objetivos:

Parte 1: Iniciar Nikto y realizar un escaneo básico
Parte 2: Usar Nikto para escanear varios servidores web
Parte 3: Investigar las vulnerabilidades de los sitios web
Parte 4: Exportar resultados de Nikto a un archivo


6.1.8 Práctica de Laboratorio - Uso del Escáner de Vulnerabilidades GVM
Tareas de Soluciones de Seguridad Protego

Continuaremos con el análisis de vulnerabilidades web pasando a la aplicación web de la tienda digital Pixel Paradise. Utilizaremos el escáner Greenbone Vulnerability Management (GVM) porque tiene funciones más avanzadas para encontrar y administrar vulnerabilidades en aplicaciones web. Algunas personas se confunden sobre la relación entre la popular herramienta OpenVas y GVM. OpenVas es un componente de GVM que se inicia a través de la interfaz de GVM.

Familiarícese con GVM en la máquina virtual (VM) de nuestro laboratorio. Lo utilizará para escanear uno de los servidores de red virtual que está alojando una aplicación web. Esperamos que pueda participar en nuestros esfuerzos para encontrar vulnerabilidades en las aplicaciones web de Pixel Paradise una vez que esté al día con esta potente herramienta.

En esta práctica de laboratorio se cumplirán los siguientes objetivos:

Parte 1: Analizar un Host en busca de vulnerabilidades
Parte 2: Explotar una vulnerabilidad detectada por GVM
